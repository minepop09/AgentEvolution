# Agent 问题处理操作系统 - 架构设计（定稿）

> 讨论日期：2026-05-19 | 更新：2026-05-19

## 核心目标

给 Agent 构建一个**可移植、可进化的问题处理框架**，类似人类的问题处理方法论 + 工具箱。

---

## ✅ 已确认决策

| 决策项 | 结论 |
|--------|------|
| 知识库查询方式 | 主动查知识库 |
| 最小可用版本 | 先只做7个问题处理模块 |
| 目标 Agent 优先级 | OpenClaw 优先接入 |

## 七大模块详解

### 1. 问题预处理模块
**职责**：拆解问题，判断类型

**输入**：用户问题
**输出**：问题类型 + 涉及领域 + 复杂度评级

---

### 2. 问题认知模块 ← 三层知识探测机制

**职责**：判断 Agent 是否具备处理此问题的专业知识

**触发条件**：模型置信度 < 80% 或知识库未命中

**三层探测流程**：
1. **第一层：置信度自评** → 置信度≥80% → 直接进问题分析
2. **第二层：知识库交叉验证** → 命中且新鲜 → 进问题分析
3. **第三层：主动探测** → 发现盲区 → 触发学习流程 → 进问题分析

---

### 3. 知识库存储模块
**职责**：领域知识存取

**存储格式**：结构化 Markdown
**查询方式**：主动查询（问题分析前主动检索）

---

### 4. 问题分析模块
**职责**：在有专业性认知的情况下深度分析

**输入**：预处理结果 + 领域知识 + 模型置信度
**输出**：问题分析报告（含方法论建议 + Todo计划）

---

### 5. 工具审核模块 ← SlowMist 安全审计

**职责**：找最优工具

**流程**：
1. 枚举候选工具列表
2. 获取网络综合评分
3. **按评分排序，取前2个**
4. **使用 SlowMist Agent Security skill 进行安全审计**
5. 审计结果处理：
   - 评分 Low → 不安装，切换到下一个候选
   - 审计通过 → 安装
6. 安装失败（最多2个） → 告知用户

---

### 6. 工具安装模块
**职责**：安装和配置工具

**约束**：只安装通过审核的工具，最多2个备选

---

### 7. Todo 计划模块
**职责**：生成可执行的任务计划

**贯穿整个流程**：每个模块输出后，Todo 计划模块更新当前任务状态

---

## 进化机制

**进化单位**：
- Skill 编排顺序（不同领域的 pipeline 组合）
- 工具权重（根据成功率动态调整优先级）

**不进化的内容**：知识本身（知识库增长 ≠ 进化）

---

## OpenClaw 优先接入

OpenClaw 作为第一个接入的目标 Agent，原因是：
- AgentHansa 生态已在使用（插件化的 skill 系统）
- 龙虾派平台有现成的自动化能力
- 可以复用已有的 MySkillPackage 迁移经验
