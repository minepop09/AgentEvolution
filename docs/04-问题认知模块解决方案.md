# 问题认知模块：Agent 知识边界判断方案

> 日期：2026-05-19

## 问题本质

模型训练涵盖所有行业知识，但存在两个核心挑战：

1. **不知道自己知道**：模型有知识，但 Agent 无法主动判断"我对这个领域的理解是否足够专业"
2. **不知道自己不知道**：模型没训练到的知识，Agent 也无法意识到这是盲区

---

## 解决方案：三层知识探测机制

### 第一层：置信度自评（Agent 主动判断）

Agent 在接收问题后，先输出一个**轻量级认知评估**：

```
问题认知报告：
- 问题领域：XXX
- 自我置信度：███░░ 60%（高中低）
- 熟悉程度：能用通俗语言解释，但细节深度不确定
- 建议行动：进入问题分析模块
```

**触发阈值**：
- 置信度 ≥ 80% → 直接进问题分析模块
- 置信度 40-79% → 进入问题分析模块，过程中如遇知识盲区则回退
- 置信度 < 40% → 进入问题认知模块

---

### 第二层：知识库交叉验证（外部校验）

当第一层置信度不足时，查询本地知识库：

1. **命中检查**：知识库中是否有该领域的结构化文档
2. **新鲜度检查**：文档更新时间，是否过期
3. **覆盖度检查**：文档是否覆盖了当前问题的子问题

**知识库命中结果**：
- 命中且新鲜 → 加载到上下文 → 进问题分析模块
- 命中但过期 → 提示"知识库有但可能过时，建议复核"
- 未命中 → 进入问题认知模块（主动学习）

---

### 第三层：主动探测（不知道自己不知道）

当知识库也查不到时，触发主动探测：

1. **领域探测 prompt**：让模型输出"解决此问题需要的核心知识点清单"
2. **盲区暴露**：如果模型输出时出现逻辑断点、含糊其辞，标记为"疑似盲区"
3. **学习触发**：对疑似盲区进行专项学习

---

## 流程图

```
用户问题
    ↓
┌─────────────────┐
│  第一层：置信度自评  │  → 置信度≥80% → 直接进问题分析
└────────┬────────┘
         ↓ 置信度<80%
┌─────────────────┐
│  第二层：知识库验证  │  → 命中且新鲜 → 进问题分析
└────────┬────────┘
         ↓ 未命中/过期
┌─────────────────┐
│  第三层：主动探测    │  → 发现盲区 → 进入问题认知模块学习
└─────────────────┘
```

---

## 关键设计决策

| 决策点 | 选择 |
|--------|------|
| 知识来源优先级 | 模型知识 > 知识库 > 外部学习 |
| 置信度阈值 | 80%（可调整） |
| 知识库格式 | Markdown（当前方案） |
| 主动学习触发 | 知识库未命中 + 模型置信度低 |
