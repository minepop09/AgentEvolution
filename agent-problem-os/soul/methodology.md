# 问题处理方法论

## 核心原则

1. **先分析后行动** - 不贸然动手，先深度理解问题
2. **方法论优先** - 根据问题类型选择合适的解决方法论
3. **工具辅助** - 通过工具审核找到最优解
4. **持续学习** - 不懂就学，知识库积累

## 七大模块

### 问题预处理模块
- 拆解问题为原子子问题
- 判断领域分类（技术类/产品类/分析类/创意类）
- 评估复杂度（简单/中等/复杂/超复杂）

### 问题认知模块（三层知识探测）
1. **第一层：置信度自评** → ≥80% 直接跳过
2. **第二层：知识库验证** → 命中且新鲜则跳过
3. **第三层：主动探测** → 发现盲区才触发学习

### 知识库存储模块
- 将学习的知识以Markdown格式存储
- 按领域分类存储

### 问题分析模块
- 根据问题类型选择方法论
- 深度分析每个子问题
- 生成Todo计划

### 工具审核模块
- 枚举候选工具（取评分前2）
- SlowMist安全审计
- Low评分不安装，失败换备选

### 工具安装模块
- 安装通过审核的工具
- 验证可用性
- 失败处理

### Todo计划模块
- 贯穿全程的任务状态管理
- 自动处理依赖关系
- 展示当前进度

## 问题类型与方法论映射

| 问题类型 | 方法论 |
|---------|--------|
| 技术类 | 结构化调试法 / 重构分析法 |
| 产品类 | 用户故事映射 / 需求优先级 |
| 分析类 | 假设验证法 / 对比分析法 |
| 创意类 | 发散收敛法 |
